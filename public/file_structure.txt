SST Front 파일 구조 분석 정리
작성 기준: 2026-05-20
목적: 현재 프로젝트에서 폴더/파일이 어떤 역할을 하고, 주요 컴포넌트가 어디에서 사용되는지 파악하기 위한 읽기용 문서

주의
- 이 문서는 구조 분석용입니다.
- 실제 코드 수정, 삭제, 이동 작업은 하지 않았습니다.
- 사용처는 import 검색과 AppRoutes.jsx 라우팅 기준으로 정리했습니다.

5. 현재 활성 파일: CommunityLifeCard.jsx 상세

파일
src/components/modules/community/life/CommunityLifeCard.jsx

역할
- 인생거리 목록에서 게시글 1개를 보여주는 카드 UI
- 썸네일, 지역, 작성자, 제목, 설명, 테마, 좋아요 수, 댓글 수, 조회 수를 표시
- 카드 클릭 시 상세 페이지 이동은 부모가 넘겨준 onClick으로 처리
- 좋아요 버튼 클릭 시 onToggleLike(post.id) 호출

props
├─ post
│  ├─ id
│  ├─ thumbnail
│  ├─ title
│  ├─ description
│  ├─ author
│  ├─ region
│  ├─ regDt
│  ├─ themes
│  ├─ wishCnt
│  ├─ commentCnt
│  └─ viewCnt
│
├─ liked
│  - 현재 사용자가 좋아요한 글인지 여부
│
├─ onClick
│  - 카드 전체 클릭 이벤트
│
└─ onToggleLike
   - 좋아요 버튼 클릭 이벤트

실제 사용처
src/pages/showcase/life/CommunityLife.jsx

사용 흐름
/showcase/life
└─ CommunityLife.jsx
   ├─ API /community 호출
   ├─ catCd: CMM001로 인생거리 목록 조회
   ├─ 서버 데이터를 카드용 post 객체로 매핑
   ├─ posts.map(...)
   └─ CommunityLifeCard 렌더링
      ├─ liked={!!likedPosts[post.id]}
      ├─ onToggleLike={toggleLike}
      └─ onClick={() => navigate(`/showcase/life/view/${post.id}`)}

참고
- CommunityLifeCard 내부 주석 일부가 깨져 있음.
- UI에서 사용하는 텍스트/주석 인코딩이 깨진 파일들이 여러 곳에 있음.
- 이 문서 작성 과정에서는 수정하지 않음.

14. 결론

현재 프로젝트는 완전히 무질서한 상태라기보다는, 새 구조와 과거 구조가 동시에 남아 있는 상태에 가깝다.

큰 흐름은 다음과 같다.
├─ src/pages
│  - 라우트 단위 화면
│
├─ src/components/modules
│  - 도메인별 재사용 컴포넌트
│
├─ src/components/common
│  - 범용 공통 컴포넌트
│
├─ src/components/card
│  - 메인 페이지 중심 카드 컴포넌트
│
└─ src/layouts/admin/tailadmin
   - 관리자 템플릿 및 일부 실제 관리자 페이지

가장 주의할 부분은 다음이다.
├─ 같은 이름의 컴포넌트가 여러 위치에 있음
├─ 관리자 페이지가 두 폴더에 나뉘어 있음
├─ 페이지네이션 계열이 여러 개임
├─ List.jsx와 _List.jsx가 같이 남아 있음
├─ API 인스턴스가 src/api와 src/utils에 나뉘어 있음
└─ 한글 인코딩이 깨진 파일이 있음
